Hôm nay là Thứ 2 (08/06), ngày đầu tiên của W9 với chủ đề Deliver Smartly[cite: 2, 57]. Trọng tâm của ngày hôm nay là thiết lập nền móng tự động hóa, hướng tới mục tiêu cluster được quản lý hoàn toàn bằng GitOps (ArgoCD sync) và tuyệt đối không apply manifest thủ công nữa[cite: 11, 12, 66, 67].

## 1. Tự động hóa với GitHub Actions (CI/CD)
Để luồng code đi mượt mà và an toàn, team sẽ áp dụng chiến lược chạy pipeline theo từng bước:
* **plan-on-PR:** Khi một thành viên tạo Pull Request (PR), hệ thống chỉ chạy kiểm tra (plan) để team nhìn trước những thay đổi hạ tầng sẽ xảy ra[cite: 6, 61].
* **apply-on-merge:** Chỉ khi PR được review và gộp (merge) vào nhánh chính, hệ thống mới chính thức thực thi (apply) thay đổi đó lên môi trường[cite: 6, 61].
    * *Nguồn tham khảo:* GitHub Actions Docs — https://docs.github.com/en/actions [cite: 12, 67]

## 2. Quản lý trạng thái bằng GitOps (ArgoCD / Flux)
Thay vì đẩy code từ ngoài vào, hệ thống cluster sẽ tự động "kéo" (sync) code từ Git về. Mình cần chú ý các khái niệm kỹ thuật cốt lõi sau:
* **ArgoCD vs Flux:** Hai công cụ phổ biến nhất để thực hiện GitOps (trong W9 team ưu tiên dùng ArgoCD)[cite: 6, 61].
* **app-of-apps:** Mô hình cấu trúc thư mục giúp ArgoCD quản lý cùng lúc nhiều ứng dụng (app) thông qua một app mẹ[cite: 6, 61].
* **sync waves:** Tính năng quy định thứ tự ưu tiên khi triển khai (ví dụ: DB phải được tạo xong trước rồi mới tới Backend)[cite: 6, 61].
    * *Nguồn tham khảo:* ArgoCD Docs — https://argo-cd.readthedocs.io (start: "Getting Started" → "App of Apps") [cite: 12, 67]
    * *Nguồn tham khảo:* Flux Docs (alternative to ArgoCD) — https://fluxcd.io/flux [cite: 12, 67]
    * *Nguồn tham khảo:* GitOps Principles (OpenGitOps) — https://opengitops.dev [cite: 12, 67]

## 3. Chiến lược Hoàn tác (Rollback)
Đây là thay đổi lớn nhất về tư duy vận hành của team:
* Cấm dùng lệnh `kubectl rollout undo` để rollback trực tiếp trên cluster[cite: 6, 61]. 
* Nếu có lỗi, mọi thao tác hoàn tác phải được thực hiện bằng lệnh `git revert` trên source code[cite: 6, 61]. GitOps controller sau đó sẽ tự động đồng bộ lại trạng thái cũ cho cluster.
